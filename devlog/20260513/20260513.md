#### Asymmetric migration woes

In the previous devlog, I mentioned that alpha does not always run to 1. In cases where the habitat suitability map splits the range, alpha could optimize on zero if there are coalescences between samples on either side of the split. This scenario gave me too much confidence. In reality, habitat suitability maps which concentrate the ancestral location distributions are very likely to run to 1. Capping at 1 may be a band-aid to a larger problem.

One aspect that I didn't predict is that the default migration rate ("m" in manuscript, "coefficient" in `terracotta` package) actually can run away as well. With this observation, I've started to recontextualize what the default migration rate parameter is doing - it determines the speed at which the ancestral location distributions reach the stationary distribution. If the stationary distribution is concentrated, this may make for very likely coalescence events and therefore a fast migration rate is preferred. As long as the ancestral location distributions reach the stationary distribution, there isn't a (significant) difference between migration rates; this also means that there is no downside to rates that are "too fast". So why didn't we see this in the previous devlog?

Take a 1D 10-deme world map where suitability increases for demes that are further to the right. If your two samples start in the same deme, then you can imagine a migration rate of 0 is best, regardless of any suitability map. Alternatively, as your samples start further apart from one another (while keeping the coalescent time the same), the optimal default migration rate increases. In the last devlog, I tested a scenario where the samples were in neighboring demes. They were close enough together to have an optimal migration rate at an intermediate value ($m=0.39$). This was lucky. Had the samples started further apart, the most concentrated location distribution that they can both be found in is the stationary distribution, so migration rates run. It's important to reiterate, there isn't a penalty to running (log-likelihood curves don't decrease). This could occur as well for a map with flat suitability, but it is just less likely as the uniform stationary distribution is often overly dispersed.
